12 Security & Admin
14 features
Data Isolation
Every server action calls requireUserId() and filters by userId
User A cannot see, edit, or delete User B's data
Shared project access explicitly granted via ProjectAccess records
API Security
Telegram webhook validates x-telegram-bot-api-secret-token header
API endpoints return 403 for free users attempting Pro features
API tokens: revoked tokens immediately rejected (lookup returns null)
Email tokens: regeneration overwrites old token (unique constraint)
Telegram link codes expire after 10 minutes
Data Integrity
Delete client → cascades to all projects, tasks, images, time entries
Delete project → cascades to tasks, images, time entries, access records
Delete task → cascades to images, files, comments
Standalone tasks survive project deletion (nullable projectId)
Admin Panel
Admin-only route protected by requireAdmin() gate
Admin dashboard: 5 stat cards (users, subscriptions, projects, tasks, clients) + paginated user table with avatar, plan, counts
